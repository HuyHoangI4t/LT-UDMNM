HƯỚNG DẪN DỰ ÁN CHATBOT TƯ VẤN TUYỂN SINH TNU
==============================================

1. GIỚI THIỆU DỰ ÁN
-------------------
Dự án là hệ thống chatbot tư vấn tuyển sinh cho Trường Đại học Tây Nguyên.

Hệ thống gồm 2 phần chính:
- Frontend React: giao diện chat cho người dùng và trang dashboard cho admin.
- Backend Laravel API: xử lý câu hỏi, tìm kiếm dữ liệu tuyển sinh, gọi Gemini AI, lưu lịch sử chat và cung cấp API thống kê/quản trị.

Các thư mục/tệp chính:
- backend/: mã nguồn Laravel API, service AI/RAG, controller, migration, seeder, command import dữ liệu.
- frontend/: mã nguồn React cho trang chat và trang admin dashboard.
- run.bat: hỗ trợ chạy nhanh backend/frontend trên Windows.
- README.md: hướng dẫn tổng quan ngắn.
- HUONG_DAN_DU_AN_CHATBOT_TUYEN_SINH.txt: tài liệu hướng dẫn chi tiết này.


2. CHỨC NĂNG HIỆN CÓ
--------------------
2.1. Trang người dùng
- Chat hỏi đáp tuyển sinh với chatbot AI.
- Hỏi về ngành đào tạo, mã ngành, tổ hợp xét tuyển, điểm chuẩn, chỉ tiêu, học phí, học bổng, ký túc xá, hồ sơ, cơ hội việc làm.
- Tư vấn chọn ngành theo sở thích, năng lực và định hướng nghề nghiệp.
- Hiển thị các câu hỏi nhanh ban đầu lấy từ API /api/faq-questions; nếu API lỗi sẽ dùng danh sách mặc định.
- Gửi câu hỏi bằng nút gửi hoặc phím Enter.
- Nhập câu hỏi bằng giọng nói nếu trình duyệt hỗ trợ SpeechRecognition.
- Hiển thị phản hồi Markdown bằng React Markdown và remark-gfm.
- Lưu tối đa 10 phiên chat gần nhất vào localStorage.
- Ghi nhớ phiên chat đang chọn bằng localStorage.
- Tạo cuộc trò chuyện mới.
- Hiển thị giờ dưới từng tin nhắn.
- Sao chép nội dung tin nhắn.
- Với tin nhắn chatbot có nút gửi lại, thích và không thích. Các nút thích/không thích hiện chỉ lưu trạng thái ở frontend, chưa gửi về backend.
- Hiển thị trạng thái "Đang suy nghĩ ..." khi đang chờ backend trả lời.
- Sau mỗi câu hỏi, frontend gọi /api/faq-questions?q=...&session_id=... để lấy câu hỏi gợi ý liên quan.

2.2. Trang admin dashboard
- Đăng nhập admin bằng API /api/login và token Laravel Sanctum.
- Lưu token đăng nhập ở frontend thông qua AuthContext.
- Xem email admin đang đăng nhập.
- Đăng xuất.
- Lọc dashboard theo ngày bắt đầu, ngày kết thúc và nhóm thời gian ngày/tuần/tháng.
- Làm mới dữ liệu dashboard.
- Xuất báo cáo CSV từ endpoint /api/dashboard/export.
- Hiển thị các chỉ số:
  + Tổng số câu hỏi.
  + Số câu hỏi hôm nay.
  + Số phiên hoạt động.
  + Thời gian phản hồi trung bình.
  + Số câu hỏi realtime trong 60 phút.
- Hiển thị biểu đồ:
  + Thống kê tổng quan.
  + Top ngành được hỏi.
  + Phân bổ intent.
  + Câu hỏi theo thời gian.
  + Ngành đang được quan tâm.
  + Phương thức xét tuyển.
  + Nền tảng truy cập.
  + Tỉnh/thành quan tâm.
- Hiển thị bảng hoạt động realtime gần nhất.
- Hiển thị phần tổng hợp xu hướng.

Lưu ý: frontend hiện chưa có màn hình CRUD riêng cho chat logs, knowledge_bases hoặc admission_majors. Các chức năng quản lý dữ liệu này đang có dưới dạng API backend.

2.3. Backend chatbot
- Nhận câu hỏi tại POST /api/chat.
- Đọc session từ header X-Session-ID; nếu không có thì tự tạo UUID.
- Phân tích câu hỏi bằng QuestionAnalyzerService để nhận diện:
  + intent: diem_chuan, hoc_phi, to_hop, chi_tieu, ma_nganh, co_hoi_viec_lam, chuong_trinh_dao_tao, hoc_bong, ky_tuc_xa, ho_so, tu_van_nganh, nganh_dao_tao, general.
  + major: tên ngành nếu nhận diện được.
  + year: năm tuyển sinh.
  + admission_method: thpt, hoc_ba, dgnl, tuyen_thang, ket_hop.
  + score: điểm của thí sinh nếu có.
  + province: tỉnh/thành nếu có.
  + profile signals: điểm mạnh, sở thích, định hướng nghề nghiệp.
- Tìm kiếm dữ liệu bằng KnowledgeSearchService:
  + Tìm trong admission_majors theo ngành/năm hoặc danh sách ngành.
  + Tìm trong knowledge_bases theo từ khóa, danh mục và loại nguồn.
  + Có thể tìm kiếm semantic bằng embedding nếu bật AI_ENABLE_EMBEDDING_SEARCH=true.
- Gọi Gemini qua AiChatService để sinh câu trả lời dựa trên context đã truy xuất.
- Cache câu trả lời bằng CHAT_CACHE_TTL_SECONDS để giảm số lần gọi AI.
- Nếu Gemini lỗi, backend trả lời dự phòng từ dữ liệu đã tìm thấy.
- Lưu lịch sử hỏi đáp vào bảng chat_logs để phục vụ thống kê.
- Trả về thêm analysis, agent steps và thông tin RAG trong response API.

2.4. Backend quản trị dữ liệu
- Đăng nhập, xem thông tin admin, đăng xuất, đăng ký admin mới.
- API chat_logs:
  + Xem danh sách phân trang.
  + Xem chi tiết một log.
  + Xóa một log.
  + Import nhiều log qua JSON.
- API knowledge_bases:
  + Danh sách có phân trang và tìm kiếm q.
  + Xem chi tiết.
  + Tạo mới.
  + Cập nhật.
  + Xóa.
- API admission_majors:
  + Danh sách có phân trang và tìm kiếm q.
  + Xem chi tiết.
  + Tạo mới.
  + Cập nhật.
  + Xóa.
- API dashboard:
  + Tổng quan.
  + Top ngành.
  + Hot majors.
  + Câu hỏi theo intent.
  + Câu hỏi theo ngày/khoảng thời gian.
  + Tỉnh/thành quan tâm.
  + Phương thức xét tuyển.
  + Nền tảng truy cập.
  + Xu hướng.
  + Realtime.
  + Export CSV.


3. CÔNG NGHỆ SỬ DỤNG
--------------------
3.1. Backend
- PHP >= 8.2.
- Laravel Framework 12.
- Laravel Sanctum: xác thực admin bằng API token.
- L5 Swagger: sinh tài liệu API.
- OpenAI PHP Laravel package: có trong composer, nhưng luồng chatbot hiện dùng cấu hình Gemini.
- Smalot PDF Parser: hỗ trợ xử lý PDF.
- Symfony DomCrawler và CSS Selector: hỗ trợ crawl/parse nội dung web.
- PHPUnit: test backend.
- MySQL hoặc SQLite tùy cấu hình .env.

3.2. Frontend
- React 18.
- React Router DOM.
- Axios.
- Bootstrap và Bootstrap Icons.
- Chart.js và react-chartjs-2.
- React Markdown và remark-gfm.
- Sass.
- React Scripts.

3.3. AI/RAG
- Gemini API để sinh câu trả lời.
- Gemini embedding model nếu bật semantic search.
- RAG lấy context từ admission_majors và knowledge_bases rồi đưa vào prompt AI.
- Cache câu trả lời theo nội dung câu hỏi, context, analysis và history gần nhất.


4. CẤU TRÚC DATABASE CHÍNH
--------------------------
4.1. chat_logs
Lưu lịch sử hỏi đáp và dữ liệu thống kê.

Cột quan trọng:
- session_id
- platform
- user_query
- bot_response
- intent
- major_name
- admission_year
- admission_method
- score
- province
- entities
- agent_steps
- retrieval_summary
- response_time
- created_at, updated_at

4.2. knowledge_bases
Lưu dữ liệu tri thức từ website, PDF hoặc nhập thủ công.

Cột thường dùng:
- category
- source_type
- title
- content
- url
- pdf_links
- image_links
- embedding
- published_at

4.3. admission_majors
Lưu dữ liệu ngành tuyển sinh đã chuẩn hóa.

Cột chính:
- year
- major_name
- major_code
- subject_groups
- score_thpt
- score_hoc_ba
- score_dgnl
- quota
- tuition_fee
- description
- career_paths
- source_url

4.4. faq_questions
Lưu câu hỏi gợi ý.

Cột chính:
- question
- category
- source_type
- knowledge_base_id

4.5. users và personal_access_tokens
Lưu tài khoản admin và token Sanctum.


5. CÀI ĐẶT BACKEND
------------------
Yêu cầu:
- PHP 8.2 trở lên.

- MySQL hoặc SQLite.

10. CÁCH SỬ DỤNG CHATBOT
------------------------
1. Mở frontend tại http://localhost:3000.
2. Nhập câu hỏi vào ô chat.
3. Bấm nút gửi hoặc Enter.
4. Trong lúc chờ, giao diện hiện "Đang suy nghĩ ...".
5. Backend phân tích câu hỏi, tìm context, gọi AI và trả câu trả lời.
6. Tin nhắn chatbot có giờ trả lời, sao chép, gửi lại, thích/không thích.
7. Sau câu trả lời, hệ thống hiển thị câu hỏi gợi ý nếu backend trả về dữ liệu phù hợp.
8. Bấm "Cuộc mới" để tạo phiên chat mới.

Ví dụ câu hỏi:
- Ngành Công nghệ thông tin lấy bao nhiêu điểm?
- Học phí ngành Kế toán là bao nhiêu?
- Ngành Thú y xét tổ hợp nào?
- Em giỏi Toán và thích lập trình thì nên học ngành nào?
- Chỉ tiêu ngành Ngôn ngữ Anh năm nay?
- Ký túc xá của trường như thế nào?
- Hồ sơ xét tuyển cần những gì?

13. LUỒNG HOẠT ĐỘNG CHATBOT
---------------------------
1. Frontend gửi POST /api/chat với message, platform, history gần nhất và header X-Session-ID.
2. Backend QuestionAnalyzerService phân tích intent/major/year/method/score/province.
3. KnowledgeSearchService tìm dữ liệu liên quan trong admission_majors và knowledge_bases.
4. Nếu bật embedding search, backend gọi embedding để tìm thêm kết quả semantic.
5. AiChatService gọi Gemini với prompt, context, history và câu hỏi.
6. Nếu AI lỗi, backend dùng fallback reply từ dữ liệu đã truy xuất.
7. Backend lưu kết quả vào chat_logs.
8. Frontend hiển thị câu trả lời.
9. Frontend gọi GET /api/faq-questions?session_id=... để lấy câu hỏi gợi ý.
10. Backend ưu tiên tìm major_name mới nhất trong chat_logs của session và tạo câu hỏi gợi ý theo ngành đó.


14. MỘT SỐ LỖI THƯỜNG GẶP
-------------------------
14.1. npm bị chặn trên PowerShell
Lỗi:
npm.ps1 cannot be loaded because running scripts is disabled

Cách xử lý nhanh:
npm.cmd install
npm.cmd start
npm.cmd run build

14.2. Backend không kết nối database
Kiểm tra .env:
- DB_CONNECTION
- DB_HOST
- DB_DATABASE
- DB_USERNAME
- DB_PASSWORD

Sau khi sửa .env:
php artisan optimize:clear
php artisan migrate

14.3. Chatbot không trả lời AI
Kiểm tra:
- GEMINI_API_KEY đã có chưa.
- Server có gọi được Gemini không.
- Log Laravel trong backend/storage/logs.

14.4. Gợi ý câu hỏi không đúng ngành
Kiểm tra:
- Bảng chat_logs có major_name không.
- Frontend có gửi session_id lên /api/faq-questions không.
- QuestionAnalyzerService có detect được tên ngành không.

14.5. Dashboard bị 401
Nếu ADMIN_API_AUTH=true, cần đăng nhập /api/login và gửi token:
Authorization: Bearer <token>

16. GHI CHÚ BẢO MẬT
-------------------
- Không commit file .env lên git.
- Không public GEMINI_API_KEY.
- Bật ADMIN_API_AUTH=true khi deploy.
- APP_DEBUG=false khi deploy.
- Nên giới hạn throttle API chat để tránh spam. Dự án hiện đã gắn throttle cho /api/chat, /api/faq-questions và /api/login.
- Backup database trước khi chạy migrate:fresh hoặc xóa dữ liệu cũ.
