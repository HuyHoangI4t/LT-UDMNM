HƯỚNG DẪN DỰ ÁN CHATBOT TUYỂN SINH TNU
======================================

1. GIỚI THIỆU DỰ ÁN
-------------------
Dự án là hệ thống chatbot tư vấn tuyển sinh cho Trường Đại học Tây Nguyên.

Hệ thống gồm:
- Frontend React: giao diện chat cho thí sinh/người dùng.
- Backend Laravel API: xử lý chat, tìm kiếm dữ liệu tuyển sinh, gọi AI Gemini, lưu lịch sử chat và thống kê dashboard.
- Database: lưu dữ liệu ngành tuyển sinh, tri thức từ website/PDF, câu hỏi gợi ý, lịch sử chat.

Thư mục chính:
- backend/: Laravel API, service AI, migration, command import dữ liệu.
- frontend/: React chat UI.
- run.bat: file hỗ trợ chạy local trên Windows.
- README.md: hướng dẫn ngắn gọn sẵn có.


2. CÁC CHỨC NĂNG CHÍNH
----------------------
2.1. Phía người dùng
- Chat hỏi đáp tuyển sinh với chatbot AI.
- Hỏi về ngành đào tạo, mã ngành, tổ hợp xét tuyển, điểm chuẩn, chỉ tiêu, học phí.
- Tư vấn chọn ngành theo sở thích, năng lực, định hướng nghề nghiệp.
- Gửi câu hỏi nhanh từ danh sách gợi ý.
- Nhập câu hỏi bằng giọng nói nếu trình duyệt hỗ trợ SpeechRecognition.
- Hiển thị câu trả lời Markdown: danh sách, bảng, liên kết.
- Lưu lịch sử chat vào localStorage, reload trang vẫn giữ được phiên chat.
- Tạo cuộc trò chuyện mới.
- Hiện giờ bên dưới từng tin nhắn.
- Sao chép nội dung tin nhắn.
- Với tin nhắn chatbot: có nút gửi lại, like, dislike.
- Hiện trạng thái "Đang suy nghĩ ..." khi chatbot đang xử lý.
- Gợi ý câu hỏi liên quan đến major_name mới nhất trong bảng chat_logs của session hiện tại.

2.2. Phía chatbot/backend
- Phân tích câu hỏi để nhận diện:
  + intent: diem_chuan, hoc_phi, to_hop, chi_tieu, ma_nganh, co_hoi_viec_lam, chuong_trinh_dao_tao, hoc_bong, ky_tuc_xa, ho_so, tu_van_nganh, nganh_dao_tao, general.
  + major_name: tên ngành như Công nghệ thông tin, Kế toán, Y khoa...
  + admission_year: năm tuyển sinh.
  + admission_method: thpt, hoc_ba, dgnl, tuyen_thang, ket_hop.
  + score: điểm của thí sinh nếu có.
  + province: tỉnh/thành nếu có.
- Truy xuất dữ liệu từ bảng admission_majors và knowledge_bases.
- Có tùy chọn semantic search bằng embedding nếu bật cấu hình.
- Gọi Gemini để sinh câu trả lời.
- Có fallback trả lời từ dữ liệu đã truy xuất nếu AI lỗi.
- Cache câu trả lời để giảm gọi API AI.
- Lưu chat vào bảng chat_logs để làm thống kê.

2.3. Quản trị và dashboard
- Đăng nhập admin bằng Laravel Sanctum.
- Xem thông tin admin hiện tại.
- Đăng xuất.
- Đăng ký admin mới.
- Quản lý chat logs: danh sách, chi tiết, xóa, import bulk.
- Quản lý knowledge_bases: CRUD dữ liệu tri thức.
- Quản lý admission_majors: CRUD dữ liệu ngành tuyển sinh.
- Dashboard:
  + Tổng số câu hỏi.
  + Câu hỏi hôm nay.
  + Số session đang hoạt động.
  + Thời gian phản hồi trung bình.
  + Intent được hỏi nhiều nhất.
  + Top ngành được hỏi.
  + Hot majors.
  + Thống kê câu hỏi theo intent.
  + Thống kê theo ngày/tuần/tháng.
  + Heatmap tỉnh/thành.
  + Thống kê phương thức xét tuyển.
  + Thống kê theo platform.
  + Xu hướng 30 ngày.
  + Realtime logs.
  + Export CSV báo cáo.


3. CÔNG NGHỆ SỬ DỤNG
--------------------

- PHP >= 8.2.
- Laravel Framework 12.
- Laravel Sanctum: xác thực admin/API token.
- L5 Swagger: tài liệu API.
- OpenAI PHP Laravel package: package AI có sẵn trong project, tuy nhiên flow hiện tại dùng cấu hình Gemini.
- Smalot PDF Parser: xử lý PDF.
- Symfony DomCrawler và CSS Selector: crawl/parse nội dung web.
- PHPUnit: test backend.
- MySQL hoặc SQLite tùy cấu hình .env.

3.2. Frontend
- React 18.
- React Router DOM.
- Axios: gọi API.
- Bootstrap và Bootstrap Icons.
- Chart.js và react-chartjs-2: biểu đồ dashboard.
- React Markdown và remark-gfm: render câu trả lời Markdown.
- Sass.
- React Scripts.

3.3. AI/RAG
- Gemini API để sinh câu trả lời.
- Gemini embedding model nếu bật embedding search.
- RAG: lấy context từ knowledge_bases và admission_majors rồi đưa vào AI.
- Cache câu trả lời bằng CHAT_CACHE_TTL_SECONDS.

Cột thường dùng:

- title
- content
- url

- image_links

- published_at

4.3. admission_majors
Lưu dữ liệu ngành tuyển sinh đã chuẩn hóa.

Cột chính:

- major_name

- subject_groups
- score_thpt
- score_hoc_ba
- score_dgnl

10. CÁCH SỬ DỤNG CHATBOT
------------------------
1. Mở frontend.
2. Nhập câu hỏi vào ô chat.
3. Bấm nút gửi hoặc Enter.
4. Trong lúc chờ, giao diện hiện "Đang suy nghĩ ...".
5. Backend phân tích câu hỏi, tìm context, gọi AI và trả câu trả lời.
6. Tin nhắn chatbot có:
   - Giờ trả lời.
   - Sao chép.
   - Gửi lại.
   - Like/dislike.
7. Bên dưới chat có câu hỏi gợi ý liên quan đến major_name mới nhất trong chat_logs.
8. Bấm "Cuộc mới" để tạo phiên chat mới.

Ví dụ câu hỏi:
- Ngành Công nghệ thông tin lấy bao nhiêu điểm?
- Học phí ngành Kế toán là bao nhiêu?
- Ngành Thú y xét tổ hợp nào?
- Em giỏi Toán và thích lập trình thì nên học ngành nào?
- Chỉ tiêu ngành Ngôn ngữ Anh năm nay?

12. LUỒNG HOẠT ĐỘNG CHATBOT
---------------------------
1. Frontend gửi POST /api/chat với:
   - message
   - platform
   - history gần nhất
   - header X-Session-ID
2. Backend QuestionAnalyzerService phân tích intent/major/year/method/score/province.
3. KnowledgeSearchService tìm dữ liệu liên quan trong admission_majors và knowledge_bases.
4. AiChatService gọi Gemini với prompt + context + history + câu hỏi.
5. Backend lưu kết quả vào chat_logs.
6. Frontend hiện câu trả lời.
7. Frontend gọi GET /api/faq-questions?session_id=...
8. Backend tìm major_name mới nhất trong chat_logs của session và trả câu hỏi gợi ý liên quan ngành đó.
